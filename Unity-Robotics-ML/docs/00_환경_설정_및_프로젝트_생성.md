## 3. 패키지 설치

### 3.1 패키지 목록

| 패키지 | 버전/소스 | 설치 방식 | 용도 |
|--------|----------|----------|------|
| `com.unity.ml-agents` | v4.0.0 (로컬) | `file:///` URI | ML-Agents SDK |
| `com.unity.robotics.ros-tcp-connector` | 최신 (git) | UPM git URL | ROS ↔ Unity TCP 통신 |
| `com.unity.robotics.urdf-importer` | 최신 (git) | UPM git URL | URDF 로봇 모델 임포트 |

### 3.2 manifest.json 수동 편집 (실제 적용)

최종 manifest.json은 다음과 같습니다. ML-Agents는 상대 경로(`file:`)를 사용했습니다.

```json
{
  "dependencies": {
    "com.unity.ml-agents": "file:../../../Users/Administrator/Desktop/ml-agents/com.unity.ml-agents",
    "com.unity.robotics.ros-tcp-connector": "https://github.com/Unity-Technologies/ROS-TCP-Connector.git?path=/com.unity.robotics.ros-tcp-connector",
    "com.unity.robotics.urdf-importer": "https://github.com/Unity-Technologies/URDF-Importer.git?path=/com.unity.robotics.urdf-importer"
  }
}
```

> **참고**: `file:`(상대 경로)과 `file:///`(절대 URI)는 Unity에서 다르게 처리됩니다.
> 프로젝트 경로(`C:\Unity_ML-Agents\Unity-Robotics-ML`) 기준 상대 경로로 지정했습니다.

### 3.3 패키지 해결 (Resolve)

Unity가 패키지를 다운로드하고 의존성을 해결하도록 `-importPackage` 없이
프로젝트를 열기만 해도 자동 Resolve가 트리거됩니다.

```powershell
& $unityEditor -batchmode -projectPath "C:\Unity_ML-Agents\Unity-Robotics-ML" -quit
```

#### ROS-TCP-Connector 설치 상세

ROS-TCP-Connector는 다음 서브 패키지들을 포함합니다:

| 서브 패키지 | 설명 |
|-----------|------|
| `ROSConnection` | Unity ↔ ROS TCP 소켓 통신 |
| `MessageGeneration` | ROS 메시지 타입 → C# 클래스 자동 생성 |
| `ROSGeometry` | 좌표계 변환 (ROS ←→ Unity) |

#### URDF-Importer 설치 상세

URDF-Importer는 다음 기능을 제공합니다:

| 기능 | 설명 |
|------|------|
| URDF 파싱 | XML 기반 로봇 모델 파싱 |
| 메시 임포트 | .STL, .DAE 파일을 Unity Mesh로 변환 |
| ArticulationBody 변환 | URDF 관절 → Unity ArticulationBody |
| 물리 프로퍼티 | 질량, 관성, 마찰 등 자동 설정 |

#### ML-Agents 설치 상세

로컬 경로(`file:///`)로 설치된 `com.unity.ml-agents` v4.0.0:

- `Editor/`: Unity Editor 확장 (Behavior Parameters, 학습 버튼 등)
- `Runtime/`: 실행 시간 API (Agent, Academy, Sensor, Actuator 등)
- `Plugins/`: 네이티브 플러그인 (protobuf, grpc 등)
- `Tests/`: 단위 테스트

## 5. 문제 해결

### 예상 가능한 문제

#### 문제 1: Git 패키지 해결 실패
- **증상**: `failed to resolve package` 오류
- **원인**: 네트워크 문제, Git 자격 증명
- **해결**: 
  1. `git config --global http.sslVerify false`
  2. VPN/프록시 설정 확인
  3. 수동 다운로드 후 로컬 경로로 대체

#### 문제 2: Unity 버전 호환성
- **증상**: `API compatibility` 경고
- **원인**: Robotics Hub 패키지가 Unity 2020.2+ 대상이나 6000.0에서 변경된 API
- **해결**: 
  1. 스크립트 심볼 정의 확인
  2. 필요시 Unity 2022.3 LTS로 전환 (Unity Hub에서 설치)

#### 문제 3: 로컬 패키지 경로 오류
- **증상**: `package with file:/// path not found`
- **원인**: 상대 경로 또는 잘못된 URI 형식
- **해결**: 절대 경로 `file:///C:/...` 형식 사용
