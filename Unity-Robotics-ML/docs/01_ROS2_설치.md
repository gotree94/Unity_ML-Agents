# Phase 1: ROS2 Humble 설치 및 환경 설정 (VirtualBox)

## 개요

ROS2 Humble Hawksbill을 VirtualBox Ubuntu 22.04 VM에 설치하고
Unity (Windows 호스트)와 TCP 통신 환경을 구축합니다.

## ROS2 설치 방식: VirtualBox + Ubuntu 22.04

| 방식 | 장점 | 단점 | 상태 |
|------|------|------|------|
| **VirtualBox + Ubuntu 22.04** | 완전한 Linux, GUI, Host-Only 통신 | 자원 소모 | ✅ 사용 |
| ~~WSL2~~ | 가벼움 | MS 종속, 일부 제약 | ❌ 대체됨 |

## 1. VirtualBox VM 생성

### 1.1 사양

| 항목 | 값 |
|------|------|
| VM 이름 | `Ubuntu-22.04-ROS2` |
| OS 타입 | Ubuntu 22.04.5 LTS (Jammy Jellyfish) amd64 |
| CPU | 4 cores (호스트: AMD Ryzen 9 7940HS, 16코어) |
| RAM | 8192 MB (호스트: 61.7GB) |
| 디스크 | 80GB VDI 동적 할당 |
| 네트워크 1 | **Host-Only** (ROS 통신용, 192.168.56.x) |
| 네트워크 2 | NAT (인터넷 액세스) |

### 1.2 VM 생성 명령어

```powershell
$vb = "C:\Program Files\Oracle\VirtualBox\VBoxManage.exe"
$vmName = "Ubuntu-22.04-ROS2"

# VM 생성
& $vb createvm --name $vmName --ostype "Ubuntu_64" --register

# CPU/Memory 설정
& $vb modifyvm $vmName --memory 8192 --cpus 4

# 네트워크: Host-Only (ROS) + NAT (인터넷)
& $vb modifyvm $vmName --nic1 hostonly --hostonlyadapter1 "VirtualBox Host-Only Ethernet Adapter"
& $vb modifyvm $vmName --nic2 nat

# 디스크 생성 (80GB)
& $vb createmedium disk --filename "$vmName.vdi" --size 81920
& $vb storagectl $vmName --name "SATA Controller" --add sata
& $vb storageattach $vmName --storagectl "SATA Controller" --port 0 --device 0 --type hdd --medium "$vmName.vdi"

# ISO 마운트
& $vb storagectl $vmName --name "IDE Controller" --add ide
& $vb storageattach $vmName --storagectl "IDE Controller" --port 0 --device 0 --type dvddrive --medium "ubuntu-22.04.5-desktop-amd64.iso"
```

### 1.3 자동 설치 (VirtualBox 7.2 Unattended Install)

```powershell
& $vb unattended install $vmName `
    --iso="ubuntu-22.04.5-desktop-amd64.iso" `
    --user=rosuser --password=rosbot123 `
    --full-user-name="ROS User" `
    --hostname=ubuntu-ros2.local `
    --time-zone="Asia/Seoul" `
    --install-additions

& $vb startvm $vmName --type headless
```

### 1.4 VM 자격 증명

| 항목 | 값 |
|------|------|
| 사용자명 | `rosuser` |
| 비밀번호 | `rosbot123` |
| 호스트명 | `ubuntu-ros2.local` |

## 2. VM 네트워크 구성

### 2.1 Host-Only 네트워크

VirtualBox Host-Only Ethernet Adapter:
- 호스트 IP: `192.168.56.1`
- 서브넷: `192.168.56.0/24`
- DHCP: 비활성화
- VM IP: DHCP로 자동 할당 (보통 192.168.56.101~)

### 2.2 통신 구조

```
Windows 호스트 (Unity)              VirtualBox VM (Ubuntu 22.04)
┌────────────────────────┐         ┌────────────────────────────┐
│ Unity Editor           │         │ ROS2 Humble                │
│  └ ROSConnection       │◄────►   │  └ ROS-TCP-Endpoint        │
│    IP: 192.168.56.1    │  TCP    │    IP: 192.168.56.x        │
│    Port: 10000         │ :10000  │    Port: 10000             │
└────────────────────────┘         └────────────────────────────┘
```

> **Host-Only 네트워크**는 호스트와 VM 간 전용 통신을 제공하여
> 외부 간섭 없이 ROS 통신이 가능합니다.

## 3. ROS2 Humble 설치 (VM 내)

### 3.1 VM 접속

```bash
# VM 실행 중인지 확인
& "C:\Program Files\Oracle\VirtualBox\VBoxManage.exe" showvminfo "Ubuntu-22.04-ROS2" | Select-String "State"

# SSH 접속 (VM IP 확인 후)
# VM IP 확인: VM 콘솔에서 `ip addr` 또는 VirtualBox 네트워크 탭에서 확인
ssh rosuser@192.168.56.10x
```

VM 실행 후 로그인:
- **콘솔 로그인**: VirtualBox GUI에서 VM 창 열기 → `rosuser` / `rosbot123`
- **SSH 로그인**: `ssh rosuser@192.168.56.x` (비밀번호: `rosbot123`)

### 3.2 ROS2 Humble 설치 명령어

```bash
# VM 내에서 실행

# 1) Locale 설정
sudo apt update && sudo apt install -y locales
sudo locale-gen en_US en_US.UTF-8
sudo update-locale LC_ALL=en_US.UTF-8 LANG=en_US.UTF-8

# 2) ROS2 저장소 추가
sudo apt install -y software-properties-common curl gnupg lsb-release
sudo add-apt-repository -y universe
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key \
  -o /usr/share/keyrings/ros-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] http://packages.ros.org/ros2/ubuntu $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null

# 3) ROS2 Humble Desktop 설치 (~2GB 다운로드, 5~10분)
sudo apt update
sudo apt install -y ros-humble-desktop

# 4) 개발 도구
sudo apt install -y python3-rosdep python3-colcon-common-extensions python3-vcstool

# 5) 환경 설정
echo "source /opt/ros/humble/setup.bash" >> ~/.bashrc

# 6) rosdep 초기화
sudo rosdep init
rosdep update

# 7) 설치 확인
source ~/.bashrc
ros2 --version          # 예: ros2 humble
ros2 topic list         # 빈 리스트 (정상)
```

## 4. ROS-TCP-Endpoint 설치

ROS-TCP-Endpoint는 ROS ↔ Unity 간 TCP 통신을 담당하는 Python 노드입니다.

### 4.1 저장소 클론 및 설치

```bash
cd ~
git clone https://github.com/Unity-Technologies/ROS-TCP-Endpoint.git
cd ROS-TCP-Endpoint

# pip 의존성 설치
pip install -r requirements.txt

# colcon 빌드
cd ~ && mkdir -p ros2_ws/src
cp -r ROS-TCP-Endpoint ros2_ws/src/
cd ~/ros2_ws
colcon build --packages-select ros_tcp_endpoint

# 환경 설정
echo "source ~/ros2_ws/install/setup.bash" >> ~/.bashrc
```

### 4.2 실행 테스트

```bash
# 터미널 1: TCP Endpoint 실행
source ~/ros2_ws/install/setup.bash
ros2 run ros_tcp_endpoint default_server_endpoint.py
# Listening on 0.0.0.0:10000

# 터미널 2: Unity 전용 토픽 발행
source ~/ros2_ws/install/setup.bash
ros2 topic pub /unity_topic std_msgs/String "data: 'Hello from ROS2'"
```

## 5. Unity ↔ ROS 통신 테스트

### 5.1 VM IP 확인

```bash
# Host-Only 네트워크 IP 확인
ip addr show | grep "192.168.56"
# 예: inet 192.168.56.101/24
```

### 5.2 Unity 측 ROS Connection 설정

1. Unity Editor 실행 → 프로젝트 열기
2. 빈 GameObject 생성 → `ROS Connection` 컴포넌트 추가
3. 설정:
   - `ROS IP Address`: `192.168.56.101` (VM IP)
   - `ROS Port`: `10000`
4. `Connect` 버튼 클릭

### 5.3 송수신 검증

**Unity → ROS**:
- Unity C# 스크립트에서 `ROSConnection.Send()` 호출
- ROS 측: `ros2 topic echo /topic_name`

**ROS → Unity**:
- ROS: `ros2 topic pub /unity_topic std_msgs/String "data: 'Hello'" -1`
- Unity: `ROSConnection.Subscribe()`로 수신

## 6. 문제 해결

### 문제 1: VM IP를 찾을 수 없음
```bash
# VM 콘솔에서 네트워크 확인
ip addr
# 또는 DHCP 재할당
sudo dhclient eth0
```

### 문제 2: Host-Only 네트워크 연결 안됨
```powershell
# Windows 호스트에서 VM으로 ping 확인
ping 192.168.56.10x
# 방화벽 확인 (Vbox 네트워크 허용)
```

### 문제 3: ROS2 Humble 설치 중 오류
```bash
# apt 업데이트 후 재시도
sudo apt update && sudo apt upgrade -y
sudo apt install -y ros-humble-desktop --fix-missing
```

### 문제 4: TCP 포트 연결 실패
```bash
# VM 내에서 포트 수신 확인
ss -tlnp | grep 10000
# 방화벽 확인
sudo ufw status
# 필요시 포트 허용
sudo ufw allow 10000
```

| SSH 접속 | Host-Only (eth0) | TCP 22 |

## 8. Unity ROS-TCP-Connector 패키지 설치

Unity 측에서 ROS와 통신하기 위해 UPM으로 패키지를 설치합니다.

### 8.1 ROS-TCP-Connector

```json
// Packages/manifest.json
{
  "dependencies": {
    "com.unity.robotics.ros-tcp-connector": "https://github.com/Unity-Technologies/ROS-TCP-Connector.git?path=/com.unity.robotics.ros-tcp-connector",
    "com.unity.robotics.urdf-importer": "https://github.com/Unity-Technologies/URDF-Importer.git?path=/com.unity.robotics.urdf-importer"
  }
}
```

### 8.2 ROSConnection 설정

Unity 씬에서 `ROSConnectionPrefab` 또는 빈 GameObject에 `ROSConnection` 컴포넌트 추가:

| 설정 | 값 |
|------|------|
| ROS IP Address | `192.168.56.101` (VM의 Host-Only IP) |
| ROS Port | `10000` |
| Connect on Awake | ✅ true |

> 실제 VM IP는 환경에 따라 다를 수 있습니다. VM에서 `ip addr`로 Host-Only 인터페이스 IP를 확인하세요.

## 9. 실제 적용: Niryo One Joint Controller

Unity-Robotics-ML 프로젝트에서는 이 ROS2 통신을 기반으로
Niryo One 로봇암 (6-DOF + 그리퍼)을 제어합니다.

### 통신 구성

```
ROS2 VM (192.168.75.204)               Unity Editor
┌──────────────────────┐              ┌──────────────────────────┐
│ ROS-TCP-Endpoint     │  TCP:10000   │ JointController.cs       │
│  → /joint_states ◄───┼──────────────┼─── Publish (30Hz)        │
│  └─ /joint_commands ─┼──────────────┼─── Subscribe             │
└──────────────────────┘              └──────────────────────────┘
```

### 주요 파일

| 파일 | 설명 |
|------|------|
| `Assets/Scripts/RobotArm/JointController.cs` | ROS2 joint state pub/sub + drive 제어 |
| `Assets/URDF/niryo_one/niryo_one.urdf` | Niryo One URDF 모델 |
| `Packages/manifest.json` | ROS-TCP-Connector + URDF-Importer 의존성 |

### 알려진 이슈

| 문제 | 원인 | 해결 |
|------|------|------|
| **ArticulationBody가 움직이지 않음** | URDF-Importer의 `Controller.Start()`가 `forceLimit=0` 설정 | `InitializeDrive()` / `SetTarget()`에서 `drive.forceLimit = 1000f` 명시적 설정 |
| **Editor에서 joint state publish 중단** | Editor idle 시 FPS 0으로 떨어짐 | `Application.targetFrameRate = 60`, `QualitySettings.vSyncCount = 0` |
