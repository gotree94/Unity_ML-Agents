## 현재 동작 흐름

```
Play Mode
└─ PickAndPlaceController.Start()
   └─ 1초 후 BeginSequence()
      └─ NextStep() [현재 스텝 인덱스]
         ├─ IK Target이 있고 IKSolver가 있음?
         │   ├─ YES → IKSolver.Solve(ikTarget.position)
         │   │          ├─ 수렴 → float[6] 사용
         │   │          └─ 실패 → 하드코딩 각도 폴백
         │   └─ NO → 하드코딩 armJoints012/345 사용
         │
         ├─ SetJointTargets([arm 6개 + gripper 2개])
         └─ duration초 후 → NextStep() (다음 스텝)
```

## 알려진 이슈

| 이슈 | 설명 | 상태 |
|------|------|------|
| forceLimit=0 버그 | URDF-Importer Controller.Start()가 forceLimit=0으로 초기화. JointController.Awake()에서 Controller/FKRobot/JointControl 비활성화로 우회 | 해결됨 |
| Gripper mimic 미지원 | URDF `<mimic multiplier="-1"/>`가 Unity에서 무시됨. left/right를 항상 쌍으로 반대 부호 명령 필요 | 해결됨 (코드 수용) |
| Gripper 물리 관통 | gripper_left/right collision mesh 부족으로 닫을 때 서로 통과. STL collision mesh 한계 | 해결 보류 |
| IK FK 정확도 | 수학적 FK 모델이 Unity transform과 ±2mm 이내로 일치해야 함. Play Mode에서 Console 로그로 검증 필요 | 테스트 필요 |
| Joint Chain 순서 오류 | forearm_link 누락, tool_link가 구동축으로 잘못 포함됨. Inspector에서 수동 수정 필요 | 문서로 안내 완료 |
